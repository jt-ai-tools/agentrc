# AgentRC — 產品簡介

## 問題所在

AI 編碼代理程式（AI coding agents）的效率取決於它們接收到的內容。大多數儲存庫（repositories）缺乏結構化的元數據（metadata）——例如自定義指令、MCP 配置、就緒性基準（readiness baselines）——而這些正是代理程式產生準確、道地程式碼所需要的。採用 Copilot 和類似工具的團隊會遇到冷啟動問題：代理程式會產生看似合理但錯誤的程式碼，因為它們不理解儲存庫的慣例、架構或工具。

這種差距在規模化時會進一步擴大。擁有數百個儲存庫的組織無法為每個儲存庫手動撰寫指令，也沒有標準的方法來衡量儲存庫是否「AI 就緒（AI-ready）」或追蹤隨時間推移的改進情況。

## 目標對象

- **平台工程團隊**：在組織內推廣 AI 編碼工具——需要評估就緒程度、設定基準並大規模追蹤採用情況。
- **個人開發者**：希望他們的 AI 代理程式從第一天起就理解他們的儲存庫技術棧、慣例和架構。
- **工程領導層**：評估跨投資組合的 AI 就緒程度，具有可量化的成熟度等級和基於政策的合規性。

## AgentRC 的功能

AgentRC 自動化了使 AI 編碼代理程式發揮效能的準備工作：

1. **評估（Assess）** — 根據涵蓋儲存庫健康狀況（樣式、建置、測試、文檔、開發環境、程式碼品質、可觀測性、安全性）和 AI 設定（指令、MCP、代理程式、技能）的 9 柱就緒模型對任何儲存庫進行評分。獲得 1-5 級的成熟度等級。

2. **產生（Generate）** — 使用 Copilot SDK 分析儲存庫並產生量身定制的 `copilot-instructions.md` 或 `AGENTS.md` 檔案。支援單一儲存庫（Monorepo）：使用 `applyTo` glob 產生特定區域範圍的指令。

3. **評估（Evaluate）** — 藉由比較有、無指令時的 AI 回應（由評審模型評分）來衡量指令的影響。可用作 CI 門檻（gate）以防止退化。

4. **配置（Configure）** — 產生 `.vscode/mcp.json` 和 `.vscode/settings.json`，以便開發環境從一開始就為 AI 做好準備。

5. **規模化（Scale）** — 批次處理 GitHub 組織或 Azure DevOps 專案中的儲存庫。自動複製、產生並開啟 PR。為領導層產生成合的就緒性報告。

## 成熟度模型

AgentRC 的就緒性評估將儲存庫映射到 5 級成熟度模型：

| 等級 | 名稱         | 意義                                                |
| ---- | ------------ | --------------------------------------------------- |
| 1    | 功能性（Functional）   | 建置、測試、基礎工具已到位                         |
| 2    | 文件化（Documented）   | README、CONTRIBUTING、自定義 AI 指令已存在         |
| 3    | 標準化（Standardized） | CI/CD、安全政策、CODEOWNERS、可觀測性              |
| 4    | 優化（Optimized）    | 已配置 MCP 伺服器、自定義代理程式、AI 技能         |
| 5    | 自主（Autonomous）   | 在極少監管下實現完全 AI 原生開發                   |

組織可以定義 **就緒政策（readiness policies）** 來過客製化評估哪些標準、覆蓋評分元數據並設定通過率閾值——在本地或 CI 中執行。

## 構建方式

- **TypeScript CLI** (Commander.js) 帶有互動式 TUI (Ink/React)
- **VS Code 擴充功能**，具有樹狀檢視、導覽和就緒報告的網頁視圖（webview）
- **Copilot SDK**，用於使用開發者已在使用的相同模型產生指令
- 支援 **GitHub** (Octokit) 和 **Azure DevOps** (REST API) 進行批次操作和 PR 建立

## 關鍵設計決策

- **指令是產生的，而非模板化的。** AgentRC 使用 Copilot SDK 分析實際的儲存庫內容——無通用的樣板代碼。
- **就緒性是可衡量的。** 9 柱模型產生數值評分和成熟度等級，從而可以設定組織範圍的基準和 CI 門檻。
- **評估形成閉環。** 團隊可以透過可配置的通過率閾值證明指令確實改善了 AI 輸出。
- **政策驅動的合規性。** 政策是可組合的 JSON 檔案，可以簽入儲存庫或在整個組織內分發。
- **批次優先。** 每個適用於單個儲存庫的工作流程也適用於數百個儲存庫——相同的 CLI，相同的輸出格式。
